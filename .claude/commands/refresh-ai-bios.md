av-actress-blog（FANZAアフィリエイトサイトの静的サイトジェネレーター、静的HTML自動生成＋Cloudflare Pages）のメンテナンス作業です。

## 目的

`config/actresses.yaml` に登録されている女優のうち、AI紹介文キャッシュ（`data/ai_cache/general/{id}.json`）がまだ無い女優に、あなた自身が紹介文を執筆して追加する。**ANTHROPIC_API_KEY等の外部AI APIは一切使わない**（そもそもこのリポジトリにはキーが用意されていない）。紹介文はあなた自身の文章生成能力で直接書くこと。

## 手順

1. `git pull` でこのリポジトリを最新化する。
2. `config/actresses.yaml` から女優ID（`id:`フィールド）の全リストを取得する。
3. `actress/{id}.html` が実際に存在する（＝ページ生成済み）IDのみを対象候補にする。作品数不足で生成されていない女優（`manifest.json`の`skipped_actresses`に載っている、または`actress/{id}.html`が存在しない）は対象外。
4. `data/ai_cache/general/{id}.json` が既に存在するIDは対象外とする（既存ファイルは絶対に上書き・削除しない）。
5. 残った「対象IDリスト」が0件なら、何もコミットせずここで終了してよい（no-op）。件数を報告して終わる。
6. 対象が50件を超える場合は、IDが大きい（＝新しく追加された）ものから最大50件に絞り、残りは次回の実行に持ち越す旨を報告する。
7. 対象の各IDについて `actress/{id}.html` を読み、以下を抽出する:
   - 名前・よみ（`<title>`タグやページ内表記から）
   - 身長・カップ・出身地（記載があれば。無ければ触れなくてよい）
   - 出演作タイトルの雰囲気（1本でよい）
   - VR作品かどうか
8. 抽出した情報を元に、150〜200字程度の自然な日本語紹介文を執筆する。既存キャッシュのトーン・長さの実例:
   > 圧巻のKカップを誇る東京都出身の春菜はな（はるなはな）。身長162cmの堂々たるスタイルで、濃厚な絡みを軸にした企画から着衣シチュエーションものまで幅広く出演している。ハイクオリティVRや4時間超えの長尺ベストにも数多く収録されており、迫力あるボディをじっくり味わえるのが魅力。ダイナマイトバディ好きなら押さえておきたい一人。

   条件:
   - 誇張しすぎない自然な日本語にする
   - 同じ語句・言い回しの繰り返しを避け、女優ごとに文章を変える
   - 体言止めや「〜が魅力」「〜な一人」等で締める
   - プロフィール情報が無い女優は、出演作の傾向や役柄の魅力を中心に書く
9. 各紹介文を `data/ai_cache/general/{id}.json` に `{"bio": "生成した紹介文"}` の形式（UTF-8、キーは`bio`のみ）で新規作成する。
10. `data/ai_cache/general/` 以外のファイルは一切変更しない（daily-generate CIの成果物と衝突させないため）。
11. 変更を `git add data/ai_cache/general/` でステージし、`feat: AI女優紹介文キャッシュにN名分を追加`（Nは実際に追加した件数）というメッセージでコミットしてpushする。
12. pushが最新化不足で失敗した場合は `git fetch` → `git rebase origin/master` してから再度pushする。

## 注意事項

- ANTHROPIC_API_KEY等の外部AI/LLM APIへの通信は一切行わない。紹介文はすべてあなた自身がその場で執筆する。
- 既存の `data/ai_cache/general/*.json` は絶対に上書き・削除しない。
- 作業完了後、追加した人数と対象が残っているか（50件超で持ち越しがあるか）を簡潔に報告する。
